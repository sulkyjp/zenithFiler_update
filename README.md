## Latest Changes — [1.8.2] - 2026-07-14 : AIテーマ生成を大幅強化（イメージ選択チップ追加、ペイン間の調和とプロフェッショナル・カラーチューニング）

### Added
- **AIテーマ生成に「イメージ選択チップ」を追加:** イメージを文章でうまく説明できなくても、明るさ（ライト/ダーク）・色の傾向（青系・緑系など9種、色見本ドット付き）・鮮やかさ・雰囲気（高級感・サイバーなど10種）・情景/素材（深夜のカフェ・CRTターミナルなど10種）の約36個のチップから選ぶだけでテーマを生成できるようにした。自由入力と併用でき、チップの選択は明示的な配色条件として、自由入力は世界観の物語として AI に伝わるため、頭の中のイメージに近いテーマがより安定して得られる。チップのみ（自由入力なし）でも生成可能

### Changed
- **AIテーマ生成の品質を強化（ペイン間の調和・世界観の一貫性）:** 生成の指示設計を全面的に見直し、AI が色を書き始める前に「光源・素材・空気感」の世界観と、全ペインの無彩色系表面（背景・ナビペイン・ヘッダー・ポップアップ・右クリックメニュー等）に共通で混ぜる「隠し味の色相（アンダートーン）」を宣言してから配色する方式に変更した。あわせて、ベース60%（ナビペイン・クローム）／メイン30%（作業ペイン）／アクセント10% の色量コントロール、warm/cool グレーの混在禁止、「死んだグレー（無彩色）」の禁止を導入し、各ペインがひとつの世界観で調和したテーマが安定して生成されるようにした。生成結果の品質チェックにも表面色相の一貫性検査（警告のみ）を追加し、AI 応答に前置きの説明文が付いてもテーマ JSON を正しく取り出せるよう抽出処理を強化した。既存のコントラスト基準・検証しきい値は変更していない
- **AIテーマ生成に「プロフェッショナル・カラーチューニング」を最優先ルールとして追加:** 鮮烈な色の指定（蛍光緑・原色の赤など）をそのまま出力せず、色相を保ったまま彩度・明度を UI 向けに調整（減色・洗練）してから適用するよう生成指示を強化した。あわせて (1) 選択行・選択タブなど広い面積のベタ塗りにネオン級の色を使わず、鮮やかさは選択時の文字色やアイコンで表現する、(2) 文字色に純白や無彩色のグレーを使わず、背景の色相をわずかに帯びさせて一体感を出す、(3) 右側のユーティリティペイン（タブ管理・最近開いたファイル）の行背景をサイドバーと同系のわずかな明度差に収めて孤立を防ぐ — を明文化した。品質チェックにも「ネオン級のワイドフィル」「浮いた純白/無彩グレー文字」の検査（警告のみ）を追加した

> 過去の変更履歴は [Releases](https://github.com/sulkyjp/zenithFiler_update/releases) を参照してください。
