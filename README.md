## Latest Changes — [1.9.1] - 2026-07-20 : Turned Settings into a resizable, movable standalone window and added a Stickman rest scene (house, sign, chimney smoke); also improved narrow-width layouts, theme colors, and feedback input smoothness

### Changed
- **Turned the Settings screen (Control Deck) into a resizable, movable standalone window (Issue #192):** The settings screen used to open as a fixed-size panel layered inside the main window, dimming the file manager behind it — so you couldn't see a theme take effect while configuring it. It's now a standalone window you can move by dragging its title bar and freely resize from the edges and corners. The main window behind it no longer dims, so you can compare theme changes live as you make them. The window remembers its size and position and reopens the same way next time. While settings is open, the status-bar mascot (stick figure) goes on break and pauses its usual roaming animation (stopping the heavy work to greatly reduce load)
- **Added a rest sequence where the stick-figure mascot plants an "on break" sign, heads into a little house with its dog, and smoke drifts from the chimney (Issue #192):** Opening the settings window makes a small house appear on the status bar; the mascot plants an "on break" sign in the ground, then hops into the house with its dog. Once inside, round puffs of smoke rise one by one from the chimney and keep drifting the whole time settings is open. Closing settings brings them back out, kicks the sign away, and returns to its usual roaming. The sign text follows the display language (Japanese "休憩中", English "On break", and so on across all 10 languages). While resting, the walking animation of the mascot and dog is stopped and only the light smoke is drawn at a low frame rate, so the load stays small
- **The theme list's five categories (Standard through Retro & Tech) now stay on a single row even at narrower widths (related to Issue #192):** The standalone settings window is narrower than the old full-screen view, so the fifth category (Retro & Tech) could wrap onto a second row. The minimum card width was lowered (narrow cards grow taller as their text wraps) and the mini-preview bars trimmed slightly, keeping all five categories on one row at narrower widths. The settings window's default size was also widened a little so it opens with five columns from the start
- **Achievement tiles on the Challenges / Stickman pages now also keep all ten on a single row at narrower widths (related to Issue #192):** Like the five theme categories, the narrower standalone settings window caused the tenth of the ten tiers (Tier 1–10) to wrap onto a second row. The minimum card width was lowered so all ten stay on one row at narrower widths. The earned-title (moniker) list at the top of the Challenges page also used to stretch horizontally and push the ranking card off the right edge; the monikers now wrap so the card stays compact and the ranking card is always visible
- **Moved the "File type color coding" item in the settings menu into the basic-settings group (directly under "Display"):** so that "Theme" and "Theme Store" now sit next to each other, making the theme-related navigation clearer

### Fixed
- **Further reduced input sluggishness when typing feedback in the settings screen (Issue #193):** Now that settings opens as a standalone window, the status-bar mascot (stick figure) stops roaming (goes on break) while settings is open. On top of that, while the cursor is in a text field in settings, the smoke animation drifting during its break is also paused so the UI/render thread can focus entirely on typing and the live preview. The smoke resumes automatically once you leave the text field
- **Reduced a momentary flicker of the file list when renaming a file while a browser (or another app) is open (Issue #190):** After a rename the app refreshes the list itself, but a redundant refresh triggered by the file-system watcher then ran again, rebuilding the list a second time and reading as a flicker. The app now briefly suppresses that redundant watcher-driven refresh right after its own rename, eliminating the double refresh
- **Fixed the app force-quitting right after adding a new folder or file to favorites, when the item had no color tag set (Issue #189):** The favorites color-tag styling added in v1.9.0 (left-edge bar + subtle row tint) changed the untagged-row background from `Transparent` (an actual brush instance) to something that could return `null`. The "success glow" animation played after adding a favorite treats the row's background as a real brush object, so a `null` background caused an internal error that took down the whole app. Untagged rows now always return an actual transparent brush instance, fixing the crash
- **Fixed path-existence probe cache invalidation (InvalidateProbe) not discarding an in-flight probe, so a stale result could be reused right after invalidation (e.g. during cloud-sync hydration retries):** invalidation now also drops any in-flight probe so the next check does fresh I/O

## 最新の変更履歴 — [1.9.1] - 2026-07-20 : 設定画面を可変・移動できる個別ウィンドウ化＋Stickmanの休憩演出（家・看板・煙）を追加。狭幅レイアウト・テーマ色・フィードバック入力のもっさりも改善

### Changed
- **設定画面（コントロールデッキ）を、サイズ変更・移動ができる独立したウィンドウにした（Issue #192）:** これまで設定画面はメインウィンドウ内に重ねて開く固定サイズのパネルで、開くと背後のファイラーが暗くなり、せっかくテーマを設定してもその反映を確認できなかった。設定画面を独立したウィンドウに作り替え、タイトルバーのドラッグで移動、四隅・辺のドラッグで自由にリサイズできるようにした。開いても背後のメイン画面は暗くならないので、テーマの切り替え結果をその場で見比べられる。ウィンドウのサイズと位置は記憶され、次に開いたときも同じ大きさ・場所で開く。設定を開いている間は、ステータスバーのマスコット（棒人間）は休憩に入り、通常の歩き回りアニメーションを止める（重い処理を止めて負荷を大きく下げる）
- **設定を開くと、Stickman（棒人間）が「休憩中」の看板を立てて犬と一緒に家に入り、煙突から煙が立ちのぼる演出を追加した（Issue #192）:** 設定ウィンドウを開くと、ステータスバーに小さな家が現れ、Stickman が「休憩中」の立て看板を地面に突き立ててから、犬を連れて「とうっ」と家に入る。家に入ると煙突からまるい煙がひとつずつ立ちのぼり、設定を開いている間ずっと流れ続ける。設定を閉じると家から出てきて、看板を蹴り飛ばしてから通常の歩き回りに戻る。看板の文言は表示言語に合わせて切り替わる（日本語「休憩中」／英語「On break」など全10言語）。休憩中は棒人間・犬の歩行アニメーションを止め、軽い煙だけを低頻度で描くので負荷は小さい
- **テーマ一覧の5カテゴリ（スタンダード〜レトロ & テック）が、狭めの幅でも横1列に並ぶようにした（Issue #192 関連）:** 設定を個別ウィンドウ化したことで以前の全画面表示より横幅が狭くなり、5列目「レトロ & テック」が2段目に折り返してしまうことがあった。カードの下限幅を引き下げ（狭いときはカードが縦に伸びて説明文が折り返す）、ミニプレビューの帯を少し詰めることで、より狭い幅でも5カテゴリを1行に保つようにした。設定ウィンドウの既定サイズも少し広げ、初回から5列で開きやすくした
- **チャレンジ／Stickman ページの実績タイルも、狭めの幅で10個が横1列に並ぶようにした（Issue #192 関連）:** テーマ一覧の5カテゴリと同様に、設定を個別ウィンドウ化して横幅が狭くなったことで、10段階（Tier1〜10）のタイルの10個目が2行目に折り返してしまっていた。カードの下限幅を下げて、狭い幅でも10個を1行に保つようにした。あわせて、チャレンジページ上部の獲得称号（二つ名）の一覧が横に伸び続けて右側のランキングカードを画面外へ押し出していたのを、二つ名を折り返してカード幅を抑え、ランキングが常に見えるようにした
- **設定メニューの項目「ファイル種別の色分け」を基本設定グループ（「表示」の直下）へ移動:** 「テーマ」と「テーマストア」が隣接するように整理し、テーマ関連の導線を分かりやすくした

### Fixed
- **設定画面のフィードバック入力時の「もっさり感」をさらに軽減（Issue #193）:** 設定を個別ウィンドウ化したことで、設定を開いている間はステータスバーのマスコット（棒人間）の歩き回りが止まる（休憩に入る）ようになった。加えて、設定内の文字入力欄にカーソルがある間は、休憩中に流している煙のアニメーションも一時停止するようにして、入力とプレビュー描画に処理を全振りする。入力欄から離れると煙は自動で再開する
- **ブラウザ等を開いた状態でファイル名を変更するとファイラが一瞬点滅することがあったのを軽減（Issue #190）:** ファイル名変更後、アプリ自身が一覧を更新した直後に、ファイル監視（FileSystemWatcher）由来の冗長な再更新がもう一度走り、一覧が二重に再構築されて点滅して見えることがあった。自前のリネーム直後は短時間その冗長な監視リフレッシュを抑制するようにして、二重更新をなくした
- **お気に入りに新しいフォルダ・ファイルを追加した直後、タグ未設定のアイテムでアプリが強制終了することがあったのを修正（Issue #189）:** v1.9.0 で追加したお気に入りのカラータグ表示強化（左端バー＋行の淡背景）で、カラータグ未設定時の背景色を `Transparent`（実体のあるブラシ）から `null` を返しうる実装に変えてしまっていた。追加成功時の「ふわっと光る」演出アニメーションは背景色を実際のブラシとして扱う仕組みのため、背景が `null` になっていると内部でエラーが発生し、アプリごと終了してしまっていた。カラータグ未設定時も常に実体のある透明ブラシを返すようにして解消した
- **パス存在判定キャッシュの無効化（InvalidateProbe）が在フライト中の判定を消し切れず、無効化後に古い結果を再利用し得た問題を修正:** 無効化時に実行中の判定も破棄し、次回に実 I/O で再判定するようにした（クラウド同期パスのリトライ等での内部信頼性向上）
