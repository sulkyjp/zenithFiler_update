## Latest Changes — [1.9.0] - 2026-07-18 : Reworked the AI theme generation dialog; added alternate row colors, file-type coloring, and stronger favorites styling

### Added
- **Strengthened favorites' color tag display and extended custom icons to every item (Issue #188):** Built from a user-submitted report. Color tags used to render as a small dot next to the item name; they're now shown as a left-edge color bar plus a subtle tint across the whole row, making the color read at a glance. Custom icons, previously available only on folder groups (containers), can now be set on individual file and folder favorites too. How you set a color tag or custom icon hasn't changed (right-click menu)
- **Added file-type text coloring to the file list (Issue #188):** Built from a user-submitted report. File names can now be colored by category — images, videos, audio, archives, executables, documents, and code (7 built-in categories) — via Settings > File type colors (off by default). A recommended set of extension/color pairings ships out of the box, but every rule's extensions, color, and enabled state are fully editable, and you can add or remove rules freely. Just like the alternate row option, the actual displayed color is automatically adjusted against the current theme's background, so it never becomes hard to read on any theme
- **Added an "alternate row colors" option to the file list (Issue #188):** Built from a user-submitted report. Users said that in wide or tall listings it was hard to track a row across columns, so every other row can now be tinted a very subtle color (Settings > File display, off by default). Rather than a bold striped look, the shading stays subtle enough to never compete with hover/selection colors. The color isn't fixed — it's computed automatically against the current theme's background, so it looks natural whether you're on one of the 50 built-in themes, an AI-generated theme, or a theme downloaded from the Theme Store
- **Added a "Blur proper nouns (rights protection)" option to AI theme generation:** When creating a theme inspired by copyrighted works (characters, titles, brands, etc.), you can now strip all proper nouns from the generated theme name and description, keeping only the colors and mood. - **「固有名詞をぼかす」の効果とリスクを、チェックボックスの下に明記するようにした:** これまでは「固有名詞をぼかす（権利保護）」という短いラベルだけで、オン・オフで何がどう変わるのかがツールチップを開かないと分からなかった。概要入力欄の直下に移したうえで、オンのときは実在の作品・ブランド名をAIが抽象的な概念・配色に変換すること、オフのときは著作権・商標権を含むテーマが生成される可能性があり公開・共有時は注意が必要なことを、常時表示の説明文として添えた

- **AIテーマ生成で、雰囲気・情景タグを右クリックして「これは絶対に入れたくない」と指定できるようにした:** これまでは「入れたいもの」しか伝えられず、たとえば「かわいい雰囲気にしたいが、ホラーっぽさだけは絶対に避けたい」といった除外の意図をAIに伝える手段がなかった。雰囲気・情景のタグを右クリックすると**NG指定**（青背景＋打ち消し線）になり、AIはその方向をパレット・素材・光・シルエット・テーマ名・説明文のいずれにも出さず、連想される色相にも寄らないようになる。NG指定は採用したタグや自由入力より優先される。もう一度右クリックすると解除でき、採用中のタグを右クリックすればそのままNGへ切り替わる。世界観セクションには操作方法の凡例を常時表示し、各タグのツールチップにも操作方法を併記した。NG指定だけを指定して（自由入力なしで）生成することもできる。なおNG指定はテーマのタグとしては保存されない（「入れたくないもの」はテーマの性格を表さないため）
- **AIテーマ生成で「固有名詞をぼかす」をONにしたとき、テーマ名や説明文の内容が薄くなりがちだったのを改善:** 固有名詞を避けようとするあまり、AIが安全側に倒して「短く・曖昧で・無難な」説明文を書く傾向があった。AIへの指示を「マスクは削除ではなく翻訳である」という方針に作り替え、①元ネタの視覚的シグネチャ（象徴的な配色の組み合わせ・素材・光源・シルエット・時代感・感情）を洗い出す →②それぞれを固有名詞ゼロの言葉に置き換える（配色は配色のまま、素材は素材のまま、消えるのは名前だけ）→③すべてを名前と説明文に織り込む、という手順と変換例を明示した。あわせて「マスクなしで書く場合と同じ長さ・具体性・喚起力で書くこと。マスクを理由に短くしてはならない」という分量のルールを課した。権利保護（固有名詞ゼロ）はこれまでどおり堅持しつつ、元ネタの空気感がしっかり伝わる濃さを狙う

### Fixed
- **AIテーマ生成で、選択ボタンのツールチップがカーソルを左から右へ動かしたときだけ切り替わらなかったのを修正:** ツールチップがカーソルの右下に表示されるため、左から右へカーソルを動かすとツールチップ自身にカーソルが当たってしまい、隣のボタンの説明に切り替わらなかった（右から左へは正常に切り替わっていた）。ツールチップをボタンの真上に固定表示するようにして解消した。同じ問題を抱えていたAIフォルダ生成の「重視ポイント」チップもあわせて修正した
- **AIテーマ生成のタグ選択チップだけ文字が大きく表示され、選択中の文字色がテーマによっては読みづらかったのを修正:** 雰囲気・情景のチップだけ文字がほかのチップより大きく表示され、選択中もアクセント色の背景に対して文字色が追従せず、テーマによっては読みづらかった。全チップで文字サイズを統一し、選択中はアクセント背景用の文字色を使うようにした
- **AIテーマ生成で「固有名詞をぼかす」がONのままテーマ名を空欄にして生成に失敗すると、「再試行」ボタンが効かなかったのを修正:** ぼかしON時はテーマ名が任意になるが、再試行の判定だけが旧来の「テーマ名必須」のままで、押しても再生成されず入力画面に戻ってしまっていた
- **設定画面のカードの影の描画方法を見直した（Issue #184）:** 設定画面のカードに掛けている影は、カードが表示されている間ずっと、ぼかしを毎フレーム計算し直していた。影の見た目は動かないため、一度描いた結果を使い回すようにした。対象はチャレンジページの「称号授与証」「ランキング」カード、ライセンスページのカード、そして全ページ共通の「トップに戻る」ボタン。見た目は一切変わらない
- **設定のページを開いただけで、アプリ全体のアニメーションが消えてしまうことがあったのを修正（Issue #185）:** 動作の重い環境を実測して自動で演出を切る「自動軽量モード」が、速いPCでも誤って作動することがあった。作動するとタブやペインの切り替え演出・テーマの演出などが以降まとめて無効になる（切り替わった旨の通知も表示される）。原因は判定の材料に2つの誤りがあったこと。①ページの初回表示にかかる時間を「このマシンは遅い」と解釈していた。チャレンジ / Stickman のページは実績カードを多数並べる都合で初回表示にだけ時間がかかるため、この2ページを続けて開くだけで作動していた。②判定に使う時間の測り方が、ユーザーが操作を続けている間ずっと延び続ける性質のもので、スクロールしながらページを見て回るだけで「遅い」と判定されうる状態だった。①は判定材料から各画面の初回表示を除いて表示済みの状態で測った値だけを使うようにし、②は操作の影響を受けずに画面の描画時間だけを測る方式に改めた。動作の重い環境で自動的に軽くする動作は従来どおり働く
- **英語表示のとき、チャレンジの二つ名150件すべてが英語になっていなかったのを修正（Issue #182）:** 表示言語を英語にすると、二つ名が「The First Ritual（はじめの儀式）」ではなく「TimidReach」のような、意味の通らない英単語をつなげた1語で表示されていた。これは内部で Stickman の動きを識別するための符号であって、二つ名として読めるものではなかった。150件すべてを本来の英題に戻した（例:「百の契約者」→ TimidReach ではなく Contractor of the Hundred、「天眼通」→ GazingUp ではなく Divine Sight）。ドイツ語・フランス語など他の8言語は元から正しく、影響を受けていたのは英語のみ。原因は Stickman の報酬データから説明文を取り込む保守用スクリプトが、動きの識別符号を英語の二つ名として書き込んでいたことで、スクリプト側も修正し、同じ壊れ方が起きていないかを検査する仕組みを追加した
- **日本語以外の表示言語で、自動生成された二つ名に日本語のかぎ括弧が付いていたのを修正（Issue #182）:** 二つ名を囲む括弧が言語によらず「 」に固定されていたため、たとえば英語表示では「Sovereign of the Abyss」のように日本語の括弧で囲まれて表示されていた。表示言語ごとに自然な引用符を使うようにした（英語・韓国語・ポルトガル語・簡体字は “ ”、ドイツ語は „ “、フランス語・スペイン語・ロシア語は « »、日本語・繁体字は従来どおり「 」）。すでに保存されている二つ名や、端末間で同期されてくる二つ名は旧来の形式で記録されているため、どちらの形式も正しく扱えるようにしてある
- **タブをドラッグしても同一ペイン内で並べ替えができなくなっていたのを修正:** 以前のバージョンで、複数行タブ表示時に選択タブの行が最下段に移動しないようにするための専用パネル（`FixedRowTabPanel`）を導入した際、ドラッグ&ドロップの挿入位置計算処理が探している型が旧来の標準パネルのままだったため、探索が常に失敗し、ドラッグ操作自体は始まってもドロップ位置が確定できず並べ替えが機能しない状態になっていた。新しいパネルを正しく認識するように修正し、ドラッグでのタブ並べ替えを復旧した
- **日本語以外の表示言語で、AIが生成する二つ名に日本語の文法規則が適用されていたのを修正（Issue #182）:** AIへの指示が日本語向けのまま全言語に使われており、「20文字以内」「助詞『の』や『を』で終わらない」「体言で締める」といった規則が英語やドイツ語にもそのまま課されていた。特に20文字という上限は英語には短すぎ（"Contractor of the Hundred" だけで25文字）、二つ名が不自然に切り詰められる原因になっていた。日本語以外では、その言語に即した長さの目安と「単独で肩書として成立する名詞句にする」という指示に切り替えた。日本語表示のときの生成内容は従来から変わらない
- **チャレンジのバックストーリー150件が、日本語・英語以外の8言語（独・西・仏・韓・葡・露・簡体字・繁体字）で未翻訳フォールバック（英語の代用表示）だったのを解消（Issue #182）:** 原因は投入時のスクリプトが日本語・英語の2言語分の訳文しか持っていなかったため。8言語×150件、計1,200件を新たに翻訳して追加した。あわせて日本語原文から英語150件も訳し直し、直訳的で不自然だった箇所（数詞の言い回し・仏教語彙の意訳等）を全体的に磨いた
- **英語版のチャレンジ二つ名・バックストーリー計300件を、英語ネイティブ視点の校閲で全件点検し、見つかった不自然な訳文19件を修正（Issue #182）:** 意味が壊れているような重大な問題はなかったが、称号とバックストーリー本文で訳語が一致していない、比喩表現が本来の意味からわずかにずれている、文法的に据わりが悪い、といった細部を洗い出して修正した（例:「透視の初学者」の Apprentice of Transparency → Apprentice of the See-Through Eye、「百の契約者」の Contractor of the Hundred → Pactbound of the Hundred）
- **AIが生成したテーマ名にバックスラッシュ等の記号が含まれていると、テーマの保存が必ず失敗していたのを修正（Issue #177）:** AIが返す名前をそのままファイル名に使っていたため、`\` のような記号が名前に混ざると「保存先が見つかりません」というエラーで保存できなかった。ファイル名として使えない記号は自動的に補正してから保存するようにした。テーマストアからダウンロードしたテーマ名にも同じ補正を適用する
- **エクスプローラーで開いているフォルダの一覧取得が失敗すると、アプリ全体が終了してしまうことがあったのを修正（Issue #176）:** ナビペインの「エクスプローラーから取り込み」機能で使っているWindowsの内部連携が一時的に失敗すると、その場でアプリごと落ちてしまっていた。取得に失敗した場合は空の一覧として扱い、アプリの動作を継続するようにした
