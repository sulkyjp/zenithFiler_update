## Latest Changes — [1.9.3] - 2026-07-22 : Expanded the Stickman companion dog's moves to 30 with a categorized picker for dog actions and props; localized the behavior generator into 8 languages, reduced Challenges-page scroll stutter, and fixed AI Favorites Organize and WebDAV subfolder issues

### Added
- **Greatly expanded the Stickman mascot's companion dog moves (8 → 30, with new dog postures) (Issue #196).** The companion dog could previously do only 8 actions; it now has 30 across five categories: Follow & lead (circle around, zoomies, lead the way, heel…), Play (play bow, chase tail, pounce, roll over…), Tricks & acrobatics (high jump, backflip, leap, hop, dance, beg…), Wait & rest (sit & stay, stretch, sniff around, dig, scratch…), and Reactions (alert, shake off…). New dog postures were added to the renderer — begging, play bow, stretch, digging, rolling on its back, sniffing, and an upright alert stance — so the moves are visually distinct. This greatly widens what behavior generation can combine.
- **Dog actions and props can now be picked from the same categorized, live-preview picker as poses (Issue #196).** The result screen previously chose "dog action" and "prop" from plain dropdowns (with dog actions shown in English). They now use the dedicated picker introduced for poses: click a category on the left to jump to it, each item shows a localized name and short description, and the selected move is demonstrated live by the stickman and dog so you can see it before choosing. Search filtering and a "None" option are included. Choosing dog actions and props is much clearer.

### Changed
- **Reduced scroll stutter (dropped frames) on the Challenges page in Settings (Issue #184, part 1).** The Challenges page shows about 150 achievement cards, and the small vector icons inside them (the reward gift, arrows, crowns — roughly 500 in total) forced a recalculation of their drawing bounds on every scroll step, which CPU tracing identified as the main cause of dropped frames. These icons are now drawn from lightweight pre-rendered images that are generated once per color and cached (still vector quality, so the appearance and crispness are unchanged). The images are prepared in the background right after the settings window opens, so there is no wait when the page appears.
- **Localized the Stickman AI behavior generator UI and pose names into 8 languages (Issue #195).** The generator's labels revamped in v1.9.2 (vibe chips, presets, category headers, pose / dog-action / prop names and descriptions, picker dialog text) previously showed placeholder text (an English draft) in the 8 non-Japanese/English languages — German, Spanish, French, Korean, Portuguese, Russian, Simplified Chinese and Traditional Chinese. About 570 strings per language were properly translated into natural, native wording so overseas users can use the feature in their own language.

### Fixed
- **Fixed "AI Favorites Organize" always failing with "Failed to organize" (Issue #198).** The step that extracts the AI's response (a JSON array describing the favorites tree) used object-only extraction logic (`{...}`), so even when the AI correctly returned an array (`[{...}, {...}]`) it grabbed only the first element and could no longer be read as a list, failing every time. It now uses dedicated array extraction that also handles code-fenced (```) responses, replies with surrounding prose, wrapper shapes such as `{"favorites":[...]}`, and output truncated mid-response. AI-driven favorites cleanup (removing broken links, grouping into virtual folders, tagging) now works correctly.
- **Improved opening subfolders on drives mapped from WebDAV/SharePoint (Issue #199).** The existence check performed before opening a folder treated "couldn't confirm in time" (a slow, undetermined response) the same as "does not exist," so an existing folder's open action was aborted. WebDAV/SharePoint-mapped drives are slow to respond on first access, so this false negative left subfolders unopenable (fast SMB shares never hit it). The existence check is now tri-state (exists / missing / undetermined) and only aborts when the folder is *confirmed* missing; when undetermined, it proceeds to open and lets the actual load decide. Existing folders on slow network/cloud drives can now be opened.

### Changed
- **設定画面のチャレンジページで、スクロール中のカクつき（コマ落ち）を軽減した（Issue #184 第1弾）:** チャレンジページは実績カード約150枚を表示しており、カード内の小さなベクターアイコン（報酬のプレゼント・矢印・王冠など約500個）が、スクロールのたびに描画範囲の再計算を発生させてフレーム落ちの主原因になっていた（CPUトレースで特定）。これらのアイコンを、色ごとに一度だけ生成してキャッシュする軽量な描画済みイメージに置き換えた（ベクター品質のままなので見た目や鮮明さは変わらない）。生成は設定画面を開いた直後にバックグラウンドで先回りして行うため、表示時の待ちも発生しない
- **Stickman（棒人間マスコット）AI行動生成のUI・ポーズ名などを8言語で本翻訳した（Issue #195）:** v1.9.2 で刷新した行動生成の各表示（雰囲気チップ・プリセット・カテゴリ見出し・ポーズ／犬の動作／小道具の名称と説明・選択ウィンドウの文言など）は、これまで日本語・英語以外の8言語（ドイツ語／スペイン語／フランス語／韓国語／ポルトガル語／ロシア語／簡体字中国語／繁体字中国語）で暫定表示（英語の仮訳）のままだった。約570項目を各言語のネイティブ表現に本翻訳し、海外ユーザーも母国語で機能を使えるようにした

### Fixed
- **「AI お気に入り整理」が「整理に失敗しました」で必ず失敗していたのを修正（Issue #198）:** AI の応答（お気に入りツリーの JSON 配列）を取り出す処理が、オブジェクト（`{...}`）専用の抽出ロジックを使っていたため、AI が正しく配列（`[{...}, {...}]`）を返しても配列の先頭の1件だけを切り出してしまい、リストとして読み込めずに毎回失敗していた。配列を正しく取り出す専用処理に切り替え、コードフェンス（```）付き・前後に説明文が混じった応答・`{"favorites":[...]}` のようなラッパー付き応答、および出力が途中で途切れたケースにも対応した。これで AI によるお気に入りの自動整理（壊れたリンクの除去・仮想フォルダへのグルーピング・タグ付け）が正しく動くようになる
- **WebDAV／SharePoint をネットワークドライブに割り当てた環境で、配下のフォルダを開けないことがあったのを改善（Issue #199）:** フォルダを開く前に行う存在確認が、応答が遅くて時間内に確認しきれなかった（＝存在するともしないとも確定できなかった）場合に「存在しない」とみなしてしまい、実在するフォルダなのに開く操作が中断されていた。WebDAV／SharePoint 割り当てドライブは初回アクセスの応答が遅いため、この取りこぼしで配下フォルダが開けなくなっていた（応答の速い SMB 共有では発生しない）。存在確認を「実在／不在／未確定」の三状態で扱うようにし、**「不在」が確定したときだけ**中断するよう変更した（「未確定」のときはそのまま開きに行き、実際の読み込みで判断する）。応答の遅いネットワーク／クラウドドライブでも、実在するフォルダを開けるようになる
