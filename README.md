## Latest Changes — [1.4.4] - 2026-07-06 : オプション画面（テーマ/実績ページ）の堅牢性・速度改善とお気に入り/ナビペインの安定化

### Changed
- **フォルダ移動時のパス存在確認の重複I/Oを削減:** お気に入りを開く際とタブでのフォルダ移動時、同じパスに対して存在確認が二重・三重に行われていた箇所を整理し、確認済みの場合は再チェックしないようにした。あわせてフォルダ存在確認の結果を短時間（0.8秒）キャッシュし、連続した確認要求で実I/Oが繰り返し発生しないようにした。監視ソフト（EDR/AV等）でディスクI/Oが遅い環境ほど効果が大きい
- **お気に入り機能の内部コードを整理（動作に変更なし）:** お気に入りツリーの再帰的な走査処理（フラット化・検索・削除・重複チェック等）が ViewModel とドラッグ&ドロップ処理側の2箇所に分かれて重複実装されていたため、共通ユーティリティに一本化した。あわせて、AI整理ダイアログ内に重複していた「壊れたリンク削除」処理も本体の機能に統合した。今後のお気に入り機能の堅牢性・保守性向上に向けたリファクタリングの第一弾
- **お気に入りツリーの種別判定（フォルダ/ファイル/リンク切れ）を高速化:** お気に入りの各項目がフォルダかファイルか、パスが実在するかどうかの判定を、表示のたびに毎回ディスク確認していた方式から、アイコン読み込み時に一度だけ確認した結果をキャッシュして参照する方式に変更した。監視ソフト（EDR/AV等）でディスクI/Oが遅い環境で、お気に入りツリー表示時の引っかかりが軽減される。リファクタリングの第二弾（保存用データ形式・DTO変換処理の整理を含む、動作に変更なし）
- **お気に入りを開く際のパス存在確認を共通キャッシュへ統合:** お気に入りを開く際の存在確認が独自の未キャッシュ処理になっており、他機能のフォルダ移動確認と結果を共有できていなかった点を解消し、既存の短時間（0.8秒）キャッシュ機構に統合した。あわせて、クラウド（Box/SPO）のハイドレーション待ち中に例外が黙って握りつぶされていた箇所にログ出力を追加した。リファクタリングの第三弾（動作に変更なし）
- **お気に入りのドラッグ&ドロップ処理を整理（動作に変更なし）:** お気に入りツリーの並べ替え・登録処理（ツリー変形・保存・失敗時のロールバック）が画面側のドラッグ&ドロップ処理に直接書かれていたため、お気に入り機能本体に移動し、画面側はドロップ位置の判定のみを担当するように整理した。あわせて、自分自身や自分の子孫への誤ドロップを防ぐ判定をドロップ確定時にも追加し、堅牢性を高めた。リファクタリングの第四弾
- **お気に入りのExcelバックアップ生成処理を分離（動作に変更なし）:** お気に入りをExcel形式でバックアップする際のシート生成処理（罫線・列幅・ハイパーリンク等）が ViewModel に直書きされ画面制御と密結合していたため、独立したサービスへ抽出した。バックアップ生成処理単体のテストが可能になった。リファクタリングの第五弾
- **お気に入り機能のコード規約統一・重複整理（動作に変更なし）:** 非同期処理の実行漏れ検知のため、投げっぱなしの非同期呼び出しをプロジェクト標準のエラーログ記録付きヘルパーに統一した。ドラッグ中のヒント文言（「Aペイン、もしくはBペインにドロップしてください」）をハードコードから10言語対応のローカライズ文字列に変更した。使われていなかったカラータグメニューの重複コードを削除し、ピン留め/ABC順/検索結果の3リストで重複していたドラッグ開始処理を共通化した。リファクタリングの第六弾（最終）
- **左ナビペインのフォルダツリー表示を高速化・安定化:** 展開したノードを常に全て描画していたため、フォルダ数が多い階層（数千件規模）を展開するとメモリ・描画負荷が積み上がり操作が重くなっていた問題を解消。表示中の範囲だけを描画する仮想化を導入した。あわせて、フォルダアイコンをノード1件ごとに個別取得していた処理を、汎用アイコンの即時表示＋実アイコンの直列取得キューに変更し、大量フォルダ展開時のスレッド負荷を削減。ペイン切替やキー操作でのフォルダ移動が連続した際、古い展開処理が完了を待たずに残り続けていた問題も解消し、常に最新の移動先だけを追従するようにした
- **履歴ビューの表示を刷新:** 参照回数を数字だけでなく背景バッジでも表現し多い項目が一目でわかるようにした。日付グループの見出しを「今日」「昨日」＋曜日付き日付の表示に変更（ローカライズ対応）。参照日時は当日の項目は時刻のみ、それ以外は月日+時刻の簡潔な表示に変更。グループの開閉状態がフォルダ移動のたびの再構築で毎回リセットされていた問題を解消し、ユーザーが開閉した状態を維持するようにした。右クリックメニューに「履歴から削除」を追加し、不要な項目を1件ずつ削除できるようにした
- **履歴ペインの表示更新を集約:** フォルダ移動のたびに履歴一覧を即座に再取得・再構築していたため、連続したフォルダ移動で不要な再描画が繰り返されていた問題を解消。変更通知を最大500msに集約してから1回だけ更新するようにした
- **履歴データベースの保守処理を間引き:** フォルダを移動するたびに古い履歴の削除処理（DBの全件スキャン）が毎回スケジュールされていたため、24時間に1回で十分な処理が過剰に実行されていた問題を解消。あわせて、時間経過による削除に加えて件数上限（500件）による削除も追加し、長期間の利用でも履歴が無制限に肥大化しないようにした
- **インデックス検索の一覧表示・検索応答性を改善:** 登録フォルダの状態（作成中/待機中/完了・ロック状態・件数）を再取得する処理が、フォルダ数分だけ都度ディスク確認とインデックス照会を行っており、更新中に頻繁に走ることでUIが引っかかることがあった。バックグラウンドスレッドで一括取得するよう変更し、短時間の連続更新は1回にまとめるようにした。あわせて、登録件数・合計件数の取得や検索処理内部のアナライザ生成を使い回す方式に変更し、無駄なインデックス再オープンを削減した
- **オプション画面「テーマ」カテゴリの重複スキャンを解消:** Control Deck のテーマカテゴリを開くたびに、起動時にウォームアップ済みのキャッシュを無視してテーマ JSON 約50個を毎回非同期で再読み込みしていた処理を、起動時と同じキャッシュを共有する方式に統一した。あわせて、テーマ JSON のメタデータ読み取りで説明文抽出とモデル解析のために同一ファイルを2回パースしていた無駄を1回のパースに統合し、テーマ一覧に変更がない再訪時はカード一覧の再構築（クリア＋全件再追加）自体をスキップするようにした。起動時のテーマ適用処理・所要時間には影響なし
- **実績ページ（Challenges/Stickman）初回表示時の設定ファイル再読込を削減:** 実績カテゴリを初めて開く、またはチャレンジ進捗を初めて記録する際に、設定ファイル（settings.json）全体をディスクから同期的に再読み込み・再デシリアライズしていた処理を、起動時に既に適用済みのメモリ上スナップショットから読む方式に変更した。UIスレッドの同期I/Oを1箇所排除。起動シーケンス自体（`ApplyStaticFlags` の実行タイミング）には変更なし
- **オプション画面のカテゴリ切替時の重複読込を整理:** バックアップ一覧・Stickman行動一覧・Stickmanストアのマイアップロード一覧が、カテゴリを再訪するたびに無条件で再読込されていた処理を、初回のみ読込・アプリ内操作（作成/削除/DL等）があった時だけ次回再訪時に再読込する方式に統一した。あわせて、オプション画面を表示するたびにデバウンス保存の未反映分を同期的にフラッシュしていた処理をバックグラウンドスレッドへ移した
- **テーマストア関連の通信・描画コストを削減:** テーマ評価集計・コミュニティテーマ一覧の取得にキャッシュがなく、テーマカテゴリを開くたびに毎回ネットワーク通信していた処理へ5分キャッシュを追加。テーマカードの色チップ表示に使うBrushを呼び出しのたびに未凍結で再生成していた処理を、キャッシュ+凍結して再利用する方式に変更。テーマカードの右クリックメニュー（項目6個）をカードごとに個別インスタンス生成していたのを単一の共有インスタンスに変更した
- **実績カード（Challenges/Stickmanページ、計300枚相当）の表示・読込コストを軽減:** カードごとに個別保持していた影エフェクト・ホバー時のグローアニメーション・達成時の常時ループアニメーションを撤去し、静的な枠線色・背景色による表現に変更。Stickman行動定義JSON・実績アイコン画像の読込結果を共有キャッシュ化し、オプション画面を開いた直後にバックグラウンドで先読みするようにした。スティッキーアンカーの表示切替判定も、スクロール量ゼロ時や状態不変時の再計算・再描画をスキップするよう最適化した

### Fixed
- **実績データのクロスデバイス同期後、実績カードの達成状況が反映されないまま古い表示が残ることがある不具合を修正:** 同期マージ完了時にメモリ上の達成数・レベルは再計算されるが、カード一覧の再構築要求フラグが立っていなかったため、次回実績ページを開いても達成状況（未達成/達成済み表示）が更新されないことがあった
- **テーマストアのボタン操作（アップロード/ダウンロード/削除/評価等）で例外が発生した場合にアプリ全体がクラッシュする可能性があった箇所を修正:** 該当ハンドラに例外処理を追加し、失敗時はエラーログを記録した上で処理を継続するようにした
- **お気に入りを開く際、監視ソフト（EDR/AV等）でディスクI/Oが遅い環境で画面が固まって見えることがある問題を修正:** お気に入りのパス存在確認がUIスレッド上で同期実行されていたため、I/Oが極端に遅い環境では数秒単位でUI全体がブロックされ、ファイル一覧が表示されないように見えることがあった。バックグラウンドスレッドで実行するよう変更し、あわせてフォルダ存在確認がタイムアウトした際にログを残すようにした（次回同様の報告があった際の原因特定用）
- **「戻る」「進む」操作でも履歴の参照回数が加算されていた問題を修正:** 戻る/進むで表示したフォルダは履歴に記録しない設計だったが、内部の記録処理が条件分岐の外側にあったため実際には記録されてしまっていた。通常のフォルダ移動のみを記録するよう修正した
- **設定ファイル（settings.json）破損時の自動復旧をトースト通知するように改善:** これまで破損検知時にバックアップから自動復旧しても画面上には何も表示されず、ユーザーが気づけない状態だった。起動完了後（ウェルカム表示完了後）に「バックアップから復元しました」「復元に失敗したため初期化しました」を通知するようにした。あわせて、終了時の設定保存処理で、インデックス検索の対象フォルダ一覧がまだ画面に読み込まれきっていない状態（起動直後の強制終了等）で保存が走った場合に、ディスク上の既存登録内容を空で上書きしてしまう可能性があった箇所にガードを追加した
- **インデックス検索の「登録したフォルダが消える・活用されない」不具合を根本解消:** 登録フォルダの即時保存と他設定のデバウンス保存が排他制御なく同時実行されると、片方が古い内容で他方の変更を上書きしてしまうことがあった（設定保存全体を直列化して解消）。また、インデックスの内部状態ファイル（indexed_roots.json）への書き込みが非アトミックで、書き込み中のクラッシュ等により破損すると全フォルダが「未インデックス」扱いになり不要な再構築が走ることがあった（一時ファイル経由のアトミック書き込みに変更し、破損時は自動バックアップから復旧するようにした）
- **フォルダ名の前方一致による誤認識で、新規登録フォルダが実際にはインデックスされないことがある不具合を修正:** 例えば「C:\Data」を登録済みの状態で「C:\Data2」のような名前が似た別フォルダを追加すると、単純な文字列前方一致判定により「既にカバー済み」と誤認識され、実際のインデックス作成がスキップされてしまっていた（フォルダの区切り文字境界を考慮した判定に修正）
- **外部ディスプレイ取り外し時にウィンドウが画面外に取り残されることがある問題を修正:** マルチディスプレイ環境からノートPC単体に切り替えた際、最大化状態のウィンドウが検知をすり抜けるケース、モニタ切断直後の過渡状態で1回きりの判定が取り逃すケース、スリープ中の取り外し等でディスプレイ変更イベント自体が届かないケースの3つの穴を塞いだ。変更イベントの1.5秒後の再確認と、ウィンドウがアクティブになった際の安全網チェックを追加し、画面外を検知した場合は従来どおりメインディスプレイへ最大化で自動復帰する。あわせて、ウィンドウ配置プリセット適用時に解像度縮小等で保存位置が作業領域からはみ出す場合は画面内に収めるようにした
