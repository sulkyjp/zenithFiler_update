## Latest Changes — [1.8.0] - 2026-07-12 : 内蔵ターミナルの疑似コンソール真因修正とコピー&ペースト・検索・使い勝手向上

### Added
- **内蔵ターミナルにコピー＆ペーストを追加:** 右クリックで「選択中ならコピー（コピー後は選択解除）、未選択ならペースト」という Windows Terminal と同様の挙動に対応し、Ctrl+Shift+C / Ctrl+Shift+V のショートカットも追加した。クリップボード操作は既存の EDR/DLP 耐性のある STA ワーカー経由で行う
- **内蔵ターミナルに検索機能（Ctrl+F）を追加:** 画面右上に検索ボックスを表示し、次候補／前候補への移動、一致件数表示に対応。長い出力からの絞り込みが容易になる
- **内蔵ターミナルに「選択で自動コピー」設定を追加:** テキストを選択した瞬間にクリップボードへ自動コピーする挙動を、設定でオンにできるようにした（既定オフ）
- **内蔵ターミナルにスクロールバック行数の設定を追加:** 保持する行数を 1000/5000/10000/50000 から選択できるようにした（既定 5000）

### Fixed
- **特定の設定条件下でターミナル機能への入り口が一切表示されず、事実上使用不能になっていた問題を修正:** 「タブ管理ビューがどこかでアクティブな間はタブバーを隠す」設定が、タブ管理ビューを配置したナビペイン自体が非表示（閉じている）状態でも有効なまま残ってしまい、タブバー上のターミナル追加ボタンが消えていた。さらにナビペイン埋め込みのターミナルビューも、レイアウト上「配置済みだが非アクティブ」な状態では初期化が実行されないため、自動タブ生成に到達しないケースがあった。タブバー非表示の判定にナビペインの実際の表示状態を加味するように修正し、あわせて起動時のビュー生成タイミングを正確に計測できるよう診断ログを整理した
- **開発ツール等から起動した場合、内蔵ターミナルが起動元プロセスの環境変数をそのまま引き継ぎ、一部 CLI ツール（Claude Code 等）が正常に起動しないことがあった問題を修正:** ターミナルのシェルを起動する中継プロセスが、起動元固有の環境変数（セッション識別用のマーカー等）を素通しで子プロセスへ継承していたため、そのマーカーを検知する CLI ツールが「多重起動」と誤認して即座に終了することがあった。起動元によらず素のターミナルと同じ環境でシェルが立ち上がるよう、該当マーカーのみを明示的に除去するようにした（ユーザー自身が設定した環境変数には影響しない）
- **内蔵ターミナルで一部の対話型 CLI ツール（Claude Code 等）が起動直後に終了してしまう問題を修正:** ターミナルのシェルを起動する中継プロセスが `TERM` 環境変数を一切設定しておらず、対話型ツールが「非対応の端末」と判定して対話モードを開始せず即終了することがあった（バージョン確認等の非対話コマンドは影響を受けないため気づきにくい不具合だった）。一般的な端末と同じ `TERM`/`COLORTERM` を設定するようにした
- **内蔵ターミナルの疑似コンソール基盤を刷新し、対話型 CLI ツールの標準出力が端末として認識されない問題に対処:** これまで OS 標準の疑似コンソール実装を直接利用していたが、一部の対話型ツールが「出力先が端末ではない」と誤判定し、非対話モードへ自動的に切り替わって入力待ちのまま止まってしまう事例を確認した。より新しく積極的にメンテナンスされている疑似コンソール実装（Windows Terminal プロジェクト由来、同梱）を優先的に使用し、読み込めない環境では従来どおり OS 標準実装に自動的に切り替わるようにした
- **日本語を含むフォルダで cmd.exe のターミナルタブを開くと、初回プロンプトの表示が文字化けしていた問題を修正:** シェル起動後に文字コード切替コマンド（UTF-8化）を後から送り込んでいたため、cmd.exe が最初のプロンプトを描画するタイミングと競合し、初回表示だけシステム既定の文字コードのまま残ってしまっていた。シェル自身の起動引数として切替コマンドを渡す方式に変更し、対話プロンプトが一度も描画される前に確実に適用されるようにした
- **ターミナルタブを開いた直後、同じプロンプト行が複数回重複表示され、直後の入力が数行上にずれて表示される（既存の文字を上書きしてしまう）問題を修正:** タブ作成直後にペインのレイアウトが確定するまでの間、疑似コンソールへのサイズ通知が複数回・異なるサイズで送られており、その都度シェル側の画面再描画が発生して既存のプロンプト行が新しい行として重複してしまっていた。サイズ通知をレイアウト確定後の1回にまとめ、それまでの間はシェルからの出力表示を保留するようにして、重複描画が起きないようにした
- **内蔵ターミナルで対話型 CLI ツール（Claude Code 等）が「端末ではない」と誤判定して起動できず、コマンドの表示位置が数行ずれて既存の行を上書きしていた問題の真因を修正:** ターミナルの中継プロセスがシェルを起動する際、Windows の内部動作（コンソール子プロセスへの標準ハンドル自動複製）により、シェルとその子プロセスの標準入出力が疑似コンソールではなく中継プロセス自身の通信パイプに接続されてしまっていた。このため (a) 対話型ツールが出力先を端末と認識できず非対話モードに落ちて即終了する、(b) エラーメッセージが画面に表示されず内部ログにだけ漏れる、(c) 出力の一部が疑似コンソールの画面管理を迂回して表示位置の食い違い（数行上への上書き・プロンプト重複）を起こす、という3つの症状が同時に発生していた。子プロセス実測で真因を特定し、シェル生成の瞬間だけ標準ハンドルの複製元を無効化することで、シェル側が正しく疑似コンソールの入出力を受け取るようにした（Claude Code の対話画面が内蔵ターミナル内で起動することを実バイナリで確認済み）

### Changed
- **ターミナルの無料利用回数（1日3回）を消費する場面を、ユーザーが明示的にターミナルタブを追加した操作のみに限定:** 従来はアプリ起動時のナビペイン自動タブ生成や、タブ・セッションの復元処理も同じ回数にカウントされており、意図せず枠を使い切ってしまうことがあった。自動生成・復元は回数消費の対象外とし、明示操作で枠に達した場合はターミナル面内に案内を表示するようにした
- **ナビペイン・Aペイン・Bペインの3箇所のターミナルを、単一の描画エンジン（WebView2 + xterm.js）に統一:** 従来ナビペインのみ WPF 自前描画（独自 ANSI パーサ・セルバッファ）で実装されており、A/Bペインの WebView2 実装と機能差（コピー＆ペースト・リンククリック・スクロールバック挙動等）があった。共通コントロール `TerminalHostControl` に描画エンジンを集約し、3箇所とも同じ入力・描画・スクロールバック挙動になるようにした。WebView2 の起動環境（`CoreWebView2Environment`）もプロセス内で共有化し、タブごとの重複初期化を解消。あわせて JS 初期化完了前に出力が届いた場合に描画が欠落することがあった問題も、メッセージのキューイングにより解消した
- **起動後のアイドルタイミングで、ターミナルの共有 WebView2 起動環境を先行生成するようにした:** これまで初めてターミナルタブを開いた際に発生していたコールド初期化ラグを、起動直後のバックグラウンド待機処理群（トレイアイコン整理・実績同期等）と同じタイミングで前もって軽減する。UIスレッドをブロックしない非同期呼び出しのみを追加しており、起動時間への影響はない
- **ターミナルタブを開く際、PTY セッション生成と WebView2 初期化を並列化:** 従来は PTY セッション生成（中継プロセス起動＋シェル起動）が完了してから WebView2 の初期化を開始する直列処理だったため、両方のコストが積み上がって表示までのラグになっていた。A/B ペインは復元時と同じ「タブを先に生成してから PTY セッションIDを後付け」する方式に統一し、ナビペインは PTY セッション生成の開始と同時に WebView2 初期化も先行着手するようにして、体感の起動ラグを削減した
- **ターミナル設定のカラーピッカーに OK/キャンセルボタンを追加:** 従来は色を選んでも確定操作がなく、外側をクリックすると（変更が保持されたまま）閉じるだけで初見だと戸惑いやすかった。ドラッグ中のライブプレビューは維持しつつ、キャンセルで選択前の色に戻せるようにした
- **A/Bペインのターミナル設定を「即時反映」から「OKで確定・キャンセルで取消」に変更:** 従来は設定項目を変更するたびに即座にディスクへ保存されており、途中でやめる操作ができなかった。ナビペインの埋め込みターミナルと同じ、OK ボタンで確定・キャンセルボタンで変更前の状態へ戻す方式に統一した（変更中のプレビュー表示は維持）

> 過去の変更履歴は [Releases](https://github.com/sulkyjp/zenithFiler_update/releases) を参照してください。
