## Highlights

- **タブ付き 2ペイン + ナビペイン** — お気に入り・ツリー・履歴・インデックス検索に即アクセスできる 6 ビュー構成のサイドバー
- **内蔵ターミナル (ConPTY)** — ペインのフォルダへ直接 cd でき、ファイル操作とコマンドライン作業をシームレスに行き来
- **50 テーマ × 全カスタマイズ** — キーバインド・エフェクト 12 カテゴリの個別制御など、見た目も操作感も自分好みに

## Latest Changes — [1.1.0] - 2026-06-11 : 実行時パフォーマンスの全面改善（ペイン・タブ切替の高速化／低速環境の自動軽量化）

### Added
- **低速環境での演出の自動軽量化:** リモートデスクトップ・ソフトウェア描画の環境や、実際の画面切替が遅い環境（セキュリティソフト等で実効性能が落ちたPCを含む）を自動検出し、装飾的なアニメーション・演出を一括でオフにして操作の速さを優先するようにした。検出時はトーストで通知する。設定 → 演出 の「低速環境で演出を自動的に軽量化」（既定オン）で無効化できる。進捗表示（グローバー・スキャンバー）などの機能的な表示は軽量化の対象にならない

### Fixed
- **テーマのランダム抽選・ペイン個別適用時のファイル読み込みをバックグラウンド化:** テーマの即時適用前に行うテーマデータの読み込みが画面側のスレッドで実行されることがあり、ファイルアクセスが遅い環境で引っかかりの原因になっていた。事前読み込みを非同期化して解消した
- **スティックマン設定カテゴリを開く際の引っかかりを解消:** カテゴリを開くたびに行動パターンのファイル群を画面側のスレッドで読み込んでいた。読み込みをバックグラウンド化し、カテゴリ切替が即応するようにした
- **ターミナルを開く際の UI フリーズを解消:** ターミナルのセッション起動（内部プロセスの起動と接続待ち）が画面側のスレッドで実行されており、セキュリティソフト等でプロセス起動が遅い環境では操作が固まり、起動失敗時は最大5秒間アプリ全体が無反応になっていた。セッション起動を完全にバックグラウンド化し、ターミナルを開く操作が常に即応するようにした
- **テーマ切替・ワーキングセット操作時の一瞬の固まりを解消:** 操作のたびに内部データの照会で UI が短時間ブロックされることがあった。当日分の照会結果をメモリに保持するようにし、繰り返し操作でも引っかからないようにした

### Changed
- **設定画面（Control Deck）を開く際の引っかかりを軽減＋起動を微高速化:** 設定画面を開くたびに全テーマのデータを読み込む処理が画面側のスレッドで実行されていたのをバックグラウンド化した（テーマカテゴリ表示時も同様）。また、起動時に読み込んでいた更新履歴（About ページ表示用）を初回参照時の読み込みに変更し、起動処理から除去した
- **描画負荷の高い装飾効果を軽量化:** 検索バーのフォーカス発光（4モード分）を軽量モード連動にし、低速環境では自動的に無効化されるようにした。また、ファイル一覧アイコン表示のカードホバー影と検索履歴一覧のアイコン発光（リスト項目ごとに GPU 合成バッファを生成し、スクロール・ホバーのたびに作り直されていた）を撤去し、境界線・色による表現に置き換えた
- **タブ切替を高速化（タブ内容のキャッシュ）:** これまでタブを切り替えるたびに、タブ内容（ファイル一覧・ツールバー・検索バーなど）の画面部品をゼロから作り直しており、切替のたびにカクつきの原因になっていた。直近のタブの画面部品を保持して切替時は表示の入れ替えだけで済むようにし、タブ切替を大幅に軽量化した（保持は直近6タブまで。スクロール位置などの表示状態も切替後に維持されるようになった）
- **実績共有が有効な場合のアプリ終了を高速化:** 終了時に実績データの書き出し完了を最大2秒待っていたのを、使用中のアイドル時間に定期的に先行書き出しする方式に変え、終了時の待ちを最大0.5秒に短縮した。タイムアウトした場合の差分も次回起動時の同期で回収されるため、共有データは失われない
- **ペイン構成（1画面/2画面/上下分割）の切替を高速化:** 切替時に画面レイアウトの再計算が最大2回走っていたのを1回に統合し、切替中はターミナル（WebView2）の中間リサイズ描画を抑止するようにした。また、2画面復帰時に保存設定ファイルを読み直していた無駄な処理を排除し、ウイルス対策ソフト等でファイルアクセスが遅い環境でも引っかからないようにした
- **ナビペインの表示/非表示切替を高速化:** 左右ナビペインの開閉のたびに内部レイアウトを全面再構築していたため、低速な環境では切替に時間がかかっていた。ビュー構成が変わらない開閉ではレイアウトを再利用するようにし、切替を大幅に軽量化した（ビューの切替・ドラッグ＆ドロップ・プリセット適用など構成が変わる操作は従来どおり再構築する）
- **マスコットのレアイベント頻度を適正化:** ステータスバーのマスコット演出で、レアイベント（クマ出没）の発動確率を引き下げ、出現が過剰にならないようにした
- **内部: 主要UI操作の性能計測ログを追加:** ペイン構成の切替・ナビペインの再構築・タブ切替・設定画面の初回表示について、所要時間を診断ログに記録するようにした（動作が重い環境の原因分析と改善効果の実測用）

> 過去の変更履歴は [Releases](https://github.com/sulkyjp/zenithFiler_update/releases) を参照してください。
