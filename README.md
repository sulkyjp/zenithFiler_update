## Latest Changes — [1.8.6] - 2026-07-16 : Fixed AI theme generation always failing with "Theme generation failed"

### Fixed
- **Fixed AI theme generation always failing with "Theme generation failed" (Issues #179 / #180):** The tag-saving step added in v1.8.5 threw an internal exception while writing the tags back into the generated theme (re-serializing the JSON), so even though the AI response itself came back fine, the final result couldn't be applied and surfaced as an error. Because selecting any style chip always produces tags, this affected nearly every generation that used the chip feature. The JSON writing used for tag saving (and the related theme-save paths) was unified onto a safe approach. As an added safeguard, if tag writing ever fails it no longer takes down the whole generation — the untagged result is used instead of erroring out

> See [Releases](https://github.com/sulkyjp/zenithFiler_update/releases) for the full history.

## 最新の変更履歴 — [1.8.6] - 2026-07-16 : AIテーマ生成が「テーマ生成に失敗しました」で必ず失敗する問題を修正

### Fixed
- **AIテーマ生成が「テーマ生成に失敗しました」で必ず失敗する問題を修正（Issue #179 / #180）:** v1.8.5で追加したタグ保存処理で、生成したテーマにタグを書き込む際の内部処理（JSONの再生成）が例外で失敗し、AIからの応答自体は正常に得られていたのに最終結果を反映できずエラーになっていた。イメージ選択チップを選んで生成すると必ずタグが付くため、チップ機能を使うほぼ全ての生成で発生していた。タグの書き込み・保存まわりのJSON生成処理を安全な方式に統一して修正した。あわせて、万一タグの書き込みに失敗しても生成全体を巻き添えにせず、タグなしの結果を採用してエラーにしないようにした

> 過去の変更履歴は [Releases](https://github.com/sulkyjp/zenithFiler_update/releases) を参照してください。
