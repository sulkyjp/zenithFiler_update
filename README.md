## Latest Changes — [1.7.1] - 2026-07-10 : 全50テーマ視認性是正・AI生成品質刷新・OpenAI/Azure OpenAI新世代モデル対応

### Changed
- **全50テーマの視認性を規格統一:** ダークテーマで一覧の文字が過度に明るく目に刺さる、背景と文字色が同系色/同色で見えにくいといった残留課題を、コントラスト監査ツールで全テーマ横断的に検出・是正した。一覧本文・補助テキスト・バッジ・タイトルバー・アドレスバー・行罫線などの可読性を規格内に統一しつつ、各テーマの個性（色相・コンセプト）は維持した
- **AIテーマ生成の品質基準を「脱・霞み」原則4・脱・原色・カテゴリ人格に刷新:** 従来はWCAG AA最低限（4.5:1等）の指示のみで、既存50テーマに適用済みの高品質基準が反映されておらず霞んだ・原色的なテーマが生成され得た。プロンプトを全面刷新し、本文コントラスト10〜13:1帯・彩度上限・アクセントファミリー同期・Professional/Premiumのカテゴリ人格などの数値基準を注入した。出力対象の色キーもハードコードからThemeKeyRegistry連動の動的生成へ変更し、キー欠落・重複記載といった腐敗を解消。生成後は35項目の品質検証（既存テーマ監査ツールと同一基準）→AIへの自動修正リトライ→決定論的なローカル自動補正、の3段構えで検証し、結果をダイアログに品質サマリとして表示するようにした
- **AI Stickman行動生成をブラッシュアップし、より個性的で唯一無二の行動を生成できるように刷新:** 従来はプロンプトが公開するポーズが実装68種中20種のみで、AI生成は手作り報酬に比べ単調になりやすかった。全ポーズに1行説明を付けて公開し、ポーズと連動して自動描画される演出（本の積み上げ・デスク・王座・光球・投擲物）の対応表も明記。禁止していた従者（followers）・後光（aura）・浮遊高度（yOffset）も「日常系は素朴に、英雄・儀式系はリッチに」という演出予算の考え方とあわせて解禁した。唯一無二性を高めるため、既存行動の名前とポーズ構成を「重複回避リスト」として毎回AIに提示し、生成のたびにランダムな演出方向性（クリエイティブシード）を添えるようにした。生成後はポーズ・犬の動作・小道具の妥当性検証、巨大化（scale）の強制無効化、名前衝突の自動回避などをローカルで検証・補正し、結果をダイアログに品質サマリとして表示する。あわせて投擲物の種類（骨/フリスビー/ボール）を実際に描き分けられるよう配線した
- **OpenAI / Azure OpenAI で新世代モデル（GPT-5系等）が使えなかった問題に対応:** 従来はChat Completions API形式（`/chat/completions`）に固定していたため、Responses API（`/responses`）専用の新世代モデルを指定するとエラーになっていた。モデル名から適切なAPI形式を自動判定し、判定が外れた場合もエラー応答から検知して自動的にもう一方の形式へ切り替えて再送するようにした（結果は以降のリクエストに引き継がれる）。応答生成に時間のかかる推論系モデルは、タイムアウトを30秒から120秒に延長した

### Fixed
- **テーマカスタマイズに表示されていたが実際には反映されなかった「フォーカスペインテキスト色」項目を削除:** 対応するUI要素が既に他の色キーを直接参照する方式へ移行済みで、変更しても画面に反映されない状態になっていた
