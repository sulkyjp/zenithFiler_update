## Latest Changes — [1.7.3] - 2026-07-11 : 社用PC診断・設定バックアップ・テーマ書込・マクロ復旧の信頼性修正

### Fixed
- **社用PC（EDR/DLP環境）で回収したパフォーマンス診断ログの信頼性を改善:** UIハング検出用watchdogが、実際にはUIスレッドが正常応答していても、監視スレッド自身がシステム全体のCPU飽和で再スケジュールされない場合に壁時計時間の膨張を実ハングと誤検出しうる問題があった（実機ログ解析で確認）。実測をUIスレッド側のコールバック内で確定させる方式に変更し、この種の偽陽性を排除した。あわせて[UI-HANG]ログにシステム全体のCPU使用率を併記し、アプリ起因かシステム飽和起因かを切り分けやすくした
- **FileSystemWatcherのシェル変更検知およびマクロのスケジュールタイマーで、UIスレッド上の同期ファイルI/Oを排除:** シェル操作直後のフォルダ作成検知（`Directory.Exists`）と、30秒ごとのマクロスケジュールチェック（`Directory.Exists`+ファイル列挙）がいずれもUIスレッドで同期実行されており、EDR/DLP環境でI/Oが数秒かかる場合にUIをブロックしうる状態だった。いずれもバックグラウンドへ退避した
- **パフォーマンス診断（デバッグ機能）が2秒ごとにUIスレッドで同期ファイル書込していた問題を修正:** 書込をバックグラウンドのライタータスクへ委譲し、診断ログ自体がUIをブロックしないようにした。あわせてjsonlにシステム全体のCPU使用率（`sys_cpu`）を追加記録するようにした
- **設定ファイル(settings.json)復旧時の通知トーストが表示されないことがあるレース条件を修正:** 起動時、KeyBindingService/IndexServiceのバックグラウンドウォームアップ等により複数の`WindowSettings.Load()`がほぼ同時に走ると、後発の呼び出しが復旧状況を上書きし、settings.json破損復旧時の通知トーストが消えることがあった。起動時に最初の`Load()`結果のみをラッチする専用プロパティを参照するように変更し、この種のレースを構造的に解消した
- **起動時のバックアップ保守処理（古い世代削除・AIサマリ格上げ）が別々のバックグラウンドタスクとして並走し、同じ保管フォルダへの軽微な競合リスクがあった問題を修正:** キャッチアップ生成→AIサマリ格上げ→古い世代削除をセマフォゲート下で直列実行する単一処理に統合した（実行タイミング・優先度は変更なし）
- **常駐モード（トレイ格納）で使い続けると設定の自動バックアップが実質的に一切生成されなくなっていた問題を修正:** 自動バックアップの生成は「アプリの真の終了時」の1経路にしか結線されておらず、×ボタンでトレイに格納する運用では終了処理そのものに到達しないため、何日使っても`backups/`フォルダが更新されないという構造的な穴があった。設定変更を検知してから約5分後に生成する仕組みを新設し、これを主経路とした上で、トレイ格納時・アプリ終了時にもその時点の状態で即時生成するようにした（3経路の冗長化）。加えて起動時に前回の生成漏れがあれば埋め合わせるキャッチアップ処理を追加し、成功時は必ずログへ1行記録することで、今回のように無自覚のまま止まる事態を検知できるようにした
- **自動バックアップが、変更概要の対象になっていない項目（テーマ・実績・AI設定など約24種類の代表的な項目リストの対象外）だけを変更した場合はスキップされていた問題を修正:** バックアップ要否の判定に「.descの要約文生成用リスト」を流用しており、対象外項目のみの変更は"無変更"と誤判定して見送られていた。判定を設定ファイル全体のセマンティックな差分のみに一本化し、対象外項目の変更でも確実にバックアップされるようにした（要約文が対象内項目に無い場合は「その他の設定を更新」と表示）
- **設定バックアップからの復元（リカバリ）を安全化:** 復元操作は現在の`settings.json`を上書きコピーするのみで、誤った世代を選んだ場合や復元対象自体が破損していた場合に元へ戻す手段がなかった。復元の直前に必ず現在の状態を「復元前の自動保存」として退避し、復元対象のJSONを構文・構造検証してから原子的（tmp書き込み→置換）に反映するように変更した。あわせて、短時間に複数回バックアップが生成される場合（自動・トレイ格納・手動が重なる等）にファイル名衝突で世代が欠落することがあった問題も解消した
- **テーマのカスタムカラー編集・AI生成テーマ保存が、書き込み途中のクラッシュで破損しうる非アトミック書込だった問題を修正:** 一時ファイル経由の原子的書込（既存の`settings.json`と同じ方式）に統一し、テーマ削除時に残りうる一時ファイルも掃除するようにした
- **マクロ本体ファイルが破損した際、該当マクロが黙って一覧から消えていた問題を修正:** 保存のたびに保持している`.bak`世代からの復旧を試みるようにし、復旧できた場合は本体ファイルへ修復書き戻しも行う
- **設定画面の「今すぐバックアップ」ボタンを押しても何も起きないことがある問題を修正:** 自動バックアップ（設定変更5分後・起動時キャッチアップ）と直前の内容が同一の場合にスキップする判定を、トリガー種別を区別せず手動クリックにも適用していたため、ユーザーが押した時点で直近の自動バックアップと差分が無いと無言で何も生成されなかった。手動クリックは「今すぐ」の明示的な要求として、この無変更スキップの対象外にし常に新しい世代を生成するようにした。あわせて、生成できたか（`bool`）を無視して常に成功トーストを出していたViewModel側も戻り値を見て通知を出し分けるように修正した

> 過去の変更履歴は [Releases](https://github.com/sulkyjp/zenithFiler_update/releases) を参照してください。
